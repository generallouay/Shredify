The Dev Checklist

Core

⚠
Error Handling
All failure paths are explicitly handled — not assumed.
0/5 ▼
All operations that can fail are wrapped in error handling (try/catch, Result types, etc.)
Errors are surfaced meaningfully — not swallowed silently
The user or caller receives a clear, actionable signal on failure
Edge cases handled: null/empty input, timeouts, unexpected state
A global / unhandled exception handler exists as a last resort
◈
Developer Logging
Enough signal to debug any issue in any environment.
0/5 ▼
Structured logs exist for key operations and failure points
Log levels used correctly (debug / info / warn / error)
Logs include enough context to trace issues without a debugger attached
No sensitive data (credentials, PII, tokens) is logged anywhere
Logs are accessible in the deployment environment, not just local console

Source Control & Delivery

⌥
Version Control & Branching
The repo tells the full story of every change.
0/5 ▼
A branching strategy is defined and followed (e.g. main = stable, feature branches = WIP)
Commits are atomic and have meaningful messages
Secrets, credentials, and local config files are excluded from source control
An ignore file (.gitignore or equivalent) is present and appropriate for the stack
The main/default branch is protected from direct pushes
◇
Releases & Versioning
Every release is deliberate, traceable, and reproducible.
0/5 ▼
A versioning convention is defined and followed (e.g. SemVer: v1.2.3)
A tag or marker on a commit explicitly signals intent to release (not every push deploys)
Release builds are clearly distinct from development/debug builds
A changelog or release notes are maintained per release
Any release artifact is versioned and traceable back to its source commit
⟳
Automated Build & Publish
No manual steps between a commit and a shipped artifact.
0/8 ▼
A CI pipeline exists and runs automatically on push or pull request
The publish/deploy workflow is fully automated — no manual steps to ship
Release builds are triggered by a specific signal (e.g. a git version tag)
Non-release pushes build and test but do not publish or deploy
Build failures block merges or deploys — the pipeline is the gatekeeper
Pipeline configuration lives in the repo alongside the code
Dependencies and build outputs are cached in CI to speed up runs
Cache is keyed on the lockfile hash — stale cache never silently breaks the build

Security & Config

⚿
Configuration & Secrets
Config is environment-aware. Secrets never touch the codebase.
0/5 ▼
All environment-specific values are externalized — nothing hardcoded
A documented example config file exists (.env.example, appsettings.example.json, etc.)
Secrets are injected at runtime via a secure store (vault, CI secrets, env vars)
The app handles missing or invalid config gracefully at startup
Config is validated early — fail fast with a clear message if misconfigured
🔒
Security
Inputs are untrusted. Access is explicitly granted, not assumed.
0/7 ▼
All user input is validated and sanitized before use
Authentication and authorization are implemented correctly and consistently
Sensitive data is encrypted at rest and in transit
Dependencies are audited for known vulnerabilities
No secrets, tokens, or credentials are ever committed to the repo
Principle of least privilege applied to services, roles, and API keys
Rate limiting or abuse prevention exists on any externally exposed interface
⇄
API & Integration Limits
Third-party limits are known, respected, and handled gracefully.
0/6 ▼
All external API rate limits are known and documented
Rate limit errors (e.g. 429) are handled explicitly — not treated as generic failures
Retry logic is implemented with exponential backoff where appropriate
API quotas (daily/monthly) are monitored and alerted on before exhaustion
Pagination is handled correctly — no assumption that first page = full result
API keys and credentials are rotatable without redeploying the app

UX & Compliance

◎
UX & Performance
Fast, clear, and handles every state — not just the happy path.
0/7 ▼
Loading states shown during async operations — no blank or frozen UI
Skeleton screens or placeholders used for content-heavy loads
Empty states handled gracefully — meaningful message, not a blank screen
Error states are shown inline — user knows what failed and what to do next
The app is responsive and tested on relevant screen sizes / form factors
Performance baseline measured (bundle size, startup time, render time, Lighthouse)

⊙
Privacy, GDPR & Compliance
User data is handled lawfully, minimally, and transparently.
0/7 ▼
Only data that is strictly necessary is collected
A legal basis for processing personal data is established (consent, contract, etc.)
Users can request access to, correction of, or deletion of their data
Cookie consent fires before any non-essential cookies or trackers are set
A privacy policy exists and is accessible from the product
Third-party services are evaluated for GDPR compliance (data processors listed)
Data retention periods are defined — data is deleted when no longer needed

Distribution & Updates

↑
Auto-Updater
Users always run the latest version with minimal friction.
0/8 ▼
🔄

Pattern: GitHub Releases + Silent Installer

Check GitHub API on startup → compare versions → download installer to %TEMP% → run silently with /CLOSEAPPLICATIONS /RESTARTAPPLICATIONS → app exits and relaunches on its own. User sees a progress bar and three choices: Update Now, Ignore, Never ask again (persisted per version).

↓ Reference implementation below (Updater.cs + UpdateDialog.cs)
Update check runs on startup — silently, on a background thread, never blocking the UI
Version comparison is correct — current vs. latest, parsed reliably (e.g. System.Version)
User can choose: update now, ignore this time, or never ask again for this version
"Never ask again" preference is persisted per version (registry, prefs file, shared prefs, etc.)
Download happens in the background with a visible progress indicator
Installer/package runs silently and handles close → install → relaunch automatically
Update check fails gracefully — no crash, no dialog if there's no network
Release asset naming is consistent and matches what the updater constructs as the download URL
Reference Implementation custom-volume-mixer · generallouay 
using System;
using System.Diagnostics;
using System.Net;
using System.IO;
using System.Text;
using Microsoft.Win32;

namespace VolumeMixer
{
    public class UpdateInfo
    {
        public string LatestVersion { get; set; }  // e.g. "v1.2.0"
        public string DownloadUrl   { get; set; }  // release page
        public string ReleaseNotes  { get; set; }
    }

    public static class Updater
    {
        // ── Change these if you ever rename the repo ──────────────────────
        private const string Owner  = "generallouay";
        private const string Repo   = "custom-volume-mixer";
        private const string ApiUrl = "https://api.github.com/repos/" + Owner + "/" + Repo + "/releases/latest";

        // Registry key for "never ask again"
        private const string REG_KEY      = @"Software\VolumeMixer";
        private const string REG_SKIP_VER = "SkipUpdateVersion";

        // Current app version — bump this with every release
        public static readonly Version CurrentVersion = new Version(2, 0, 4);

        // Returns null if up to date, network error, or user said never.
        public static UpdateInfo CheckForUpdate()
        {
            string skipVer = ReadSkipVersion();
            try
            {
                string json = FetchJson(ApiUrl);
                string tag  = ExtractField(json, "tag_name");
                string html = ExtractField(json, "html_url");
                string body = ExtractField(json, "body");

                string numericTag = tag.TrimStart('v');
                if (!Version.TryParse(numericTag, out Version latest)) return null;
                if (latest <= CurrentVersion) return null;
                if (skipVer == tag) return null;

                return new UpdateInfo
                {
                    LatestVersion = tag,
                    DownloadUrl   = html,
                    ReleaseNotes  = body
                };
            }
            catch { return null; }  // Silently fail — no internet, rate limit, etc.
        }

        public static string GetInstallerUrl(string tag)
        {
            string ver = tag.TrimStart('v');
            return $"https://github.com/{Owner}/{Repo}/releases/download/{tag}/VolumeMixer-Setup-v{ver}.exe";
        }

        // Downloads installer to %TEMP%, runs it silently, exits the app.
        public static void DownloadAndInstall(string tag, Action<int> onProgress, Action<string> onError)
        {
            string url  = GetInstallerUrl(tag);
            string dest = Path.Combine(Path.GetTempPath(), $"VolumeMixer-Setup-{tag}.exe");
            try
            {
                using (var client = new WebClient())
                {
                    client.DownloadProgressChanged += (s, e) => onProgress?.Invoke(e.ProgressPercentage);
                    client.DownloadFileCompleted   += (s, e) =>
                    {
                        if (e.Error != null)  { onError?.Invoke(e.Error.Message); return; }
                        if (e.Cancelled)       { onError?.Invoke("Cancelled.");        return; }

                        string appExe = System.Windows.Forms.Application.ExecutablePath;
                        var psi = new ProcessStartInfo(dest,
                            "/SILENT /CLOSEAPPLICATIONS /RESTARTAPPLICATIONS /RESTARTEXIT /RELAUNCHPATH=\"" + appExe + "\"")
                        {
                            UseShellExecute = true
                        };
                        Process.Start(psi);
                        System.Windows.Forms.Application.Exit();
                    };
                    client.DownloadFileAsync(new Uri(url), dest);
                }
            }
            catch (Exception ex) { onError?.Invoke(ex.Message); }
        }

        public static void SaveSkipVersion(string version)
        {
            try
            {
                using (var key = Registry.CurrentUser.CreateSubKey(REG_KEY))
                    key.SetValue(REG_SKIP_VER, version);
            }
            catch { }
        }

        private static string ReadSkipVersion()
        {
            try
            {
                using (var key = Registry.CurrentUser.OpenSubKey(REG_KEY))
                    return key?.GetValue(REG_SKIP_VER) as string ?? "";
            }
            catch { return ""; }
        }

        // Minimal JSON field extractor (no external deps)
        private static string ExtractField(string json, string field)
        {
            string search = "\"" + field + "\"";
            int idx = json.IndexOf(search, StringComparison.Ordinal);
            if (idx < 0) return "";
            idx += search.Length;
            while (idx < json.Length && (json[idx] == ' ' || json[idx] == ':')) idx++;
            if (idx >= json.Length) return "";
            if (json[idx] == '"')
            {
                idx++;
                var sb = new StringBuilder();
                while (idx < json.Length && json[idx] != '"')
                {
                    if (json[idx] == '\\') idx++;
                    if (idx < json.Length) sb.Append(json[idx]);
                    idx++;
                }
                return sb.ToString();
            }
            int end = json.IndexOfAny(new[] { ',', '}', ']' }, idx);
            return end < 0 ? json.Substring(idx) : json.Substring(idx, end - idx).Trim();
        }

        private static string FetchJson(string url)
        {
            var req = (HttpWebRequest)WebRequest.Create(url);
            req.UserAgent = "VolumeMixer-Updater/1.0";  // GitHub API requires a UA
            req.Timeout   = 5000;
            using (var resp   = (HttpWebResponse)req.GetResponse())
            using (var stream = resp.GetResponseStream())
            using (var reader = new StreamReader(stream, Encoding.UTF8))
                return reader.ReadToEnd();
        }
    }

using System;
using System.Drawing;
using System.Drawing.Drawing2D;
using System.Windows.Forms;

namespace VolumeMixer
{
    public class UpdateDialog : Form
    {
        private readonly UpdateInfo _info;

        [System.Runtime.InteropServices.DllImport("user32.dll")]
        static extern bool ReleaseCapture();
        [System.Runtime.InteropServices.DllImport("user32.dll")]
        static extern IntPtr SendMessage(IntPtr h, int msg, int w, int l);

        public UpdateDialog(UpdateInfo info) { _info = info; BuildUI(); }

        private void BuildUI()
        {
            Text            = "Update Available";
            Size            = new Size(380, 280);
            FormBorderStyle = FormBorderStyle.None;
            StartPosition   = FormStartPosition.CenterScreen;
            BackColor       = Theme.Surface;
            DoubleBuffered  = true;
            MouseDown      += Drag;

            // Thin accent bar at the top
            var topBar = new Panel { Dock = DockStyle.Top, Height = 4, BackColor = Theme.Accent };

            var titleLbl = new Label
            {
                Text = "Update Available",
                Font = new Font("Segoe UI", 13f, FontStyle.Bold),
                ForeColor = Theme.TextPrimary, BackColor = Color.Transparent,
                AutoSize = true, Location = new Point(20, 22)
            };
            titleLbl.MouseDown += Drag;

            var closeLbl = new Label
            {
                Text = "x", Font = new Font("Segoe UI", 13f),
                ForeColor = Theme.TextSecondary, BackColor = Color.Transparent,
                AutoSize = true, Cursor = Cursors.Hand, Location = new Point(Width - 30, 18)
            };
            closeLbl.Click      += (s, e) => Close();
            closeLbl.MouseEnter += (s, e) => closeLbl.ForeColor = Theme.Accent;
            closeLbl.MouseLeave += (s, e) => closeLbl.ForeColor = Theme.TextSecondary;
            Resize += (s, e) => closeLbl.Location = new Point(Width - 30, 18);

            var verLbl = new Label
            {
                Text = "Version " + _info.LatestVersion + " is ready to install.",
                Font = new Font("Segoe UI", 9f), ForeColor = Theme.TextSecondary,
                BackColor = Color.Transparent, AutoSize = true, Location = new Point(20, 58)
            };

            string notes = string.IsNullOrWhiteSpace(_info.ReleaseNotes)
                ? "No release notes provided."
                : _info.ReleaseNotes.Length > 160
                    ? _info.ReleaseNotes.Substring(0, 157) + "..."
                    : _info.ReleaseNotes;

            var notesBox = new Label
            {
                Text = notes, Font = new Font("Segoe UI", 8.5f),
                ForeColor = Color.FromArgb(100, 100, 128),
                BackColor = Theme.Background,
                Location = new Point(20, 82), Size = new Size(340, 66), Padding = new Padding(8)
            };

            // Progress bar — hidden until download starts
            var progressBar = new ProgressBarFlat
            {
                Location = new Point(20, 158), Size = new Size(340, 4),
                Visible = false, Value = 0
            };
            var progressLbl = new Label
            {
                Text = "", Font = new Font("Segoe UI", 7.5f),
                ForeColor = Theme.TextSecondary, BackColor = Color.Transparent,
                AutoSize = true, Location = new Point(20, 166), Visible = false
            };

            // Buttons: Update Now / Ignore / Never ask again
            var updateBtn = new AccentButton  { Text = "Update Now",      Size = new Size(120, 34), Location = new Point(20,  226) };
            var ignoreBtn = new GhostButton  { Text = "Ignore",           Size = new Size(80,  34), Location = new Point(150, 226) };
            var neverBtn  = new GhostButton  { Text = "Never ask again",  Size = new Size(126, 34), Location = new Point(238, 226) };

            ignoreBtn.Click += (s, e) => Close();
            neverBtn.Click  += (s, e) => { Updater.SaveSkipVersion(_info.LatestVersion); Close(); };

            updateBtn.Click += (s, e) =>
            {
                updateBtn.Enabled = false; updateBtn.Text = "Downloading...";
                ignoreBtn.Enabled = false; neverBtn.Enabled = false;
                progressBar.Visible = true; progressLbl.Visible = true;
                progressLbl.Text = "Starting download...";

                System.Threading.ThreadPool.QueueUserWorkItem(_ =>
                {
                    Updater.DownloadAndInstall(
                        _info.LatestVersion,
                        pct => BeginInvoke(new MethodInvoker(() =>
                        {
                            progressBar.Value = pct;
                            progressLbl.Text  = "Downloading... " + pct + "%";
                        })),
                        err => BeginInvoke(new MethodInvoker(() =>
                        {
                            updateBtn.Enabled = true;  updateBtn.Text = "Update Now";
                            ignoreBtn.Enabled = true;  neverBtn.Enabled = true;
                            progressBar.Visible   = false;
                            progressLbl.ForeColor = Theme.MutedRed;
                            progressLbl.Text      = "Failed - check your connection";
                        }))
                    );
                });
            };

            Controls.Add(topBar);    Controls.Add(titleLbl);   Controls.Add(closeLbl);
            Controls.Add(verLbl);    Controls.Add(notesBox);    Controls.Add(progressBar);
            Controls.Add(progressLbl); Controls.Add(updateBtn); Controls.Add(ignoreBtn);
            Controls.Add(neverBtn);
        }

        protected override void OnPaint(PaintEventArgs e)
        {
            base.OnPaint(e);
            e.Graphics.DrawRectangle(new System.Drawing.Pen(Theme.Border, 1),
                new Rectangle(0, 0, Width - 1, Height - 1));
        }

        private void Drag(object sender, MouseEventArgs e)
        {
            if (e.Button == MouseButtons.Left) { ReleaseCapture(); SendMessage(Handle, 0xA1, 0x2, 0); }
        }
    }

    // Thin flat progress bar styled to match the theme
    public class ProgressBarFlat : Control
    {
        private int _value;
        public int Value
        {
            get => _value;
            set { _value = Math.Max(0, Math.Min(100, value)); Invalidate(); }
        }

        public ProgressBarFlat()
        {
            SetStyle(
                ControlStyles.OptimizedDoubleBuffer |
                ControlStyles.UserPaint             |
                ControlStyles.AllPaintingInWmPaint  |
                ControlStyles.SupportsTransparentBackColor, true);
            BackColor = Color.Transparent;
        }

        protected override void OnPaint(PaintEventArgs e)
        {
            Graphics g = e.Graphics;
            g.FillRectangle(new SolidBrush(Theme.Border), 0, 0, Width, Height);
            if (_value > 0)
            {
                int fillW = (int)(Width * (_value / 100f));
                using (var brush = new LinearGradientBrush(
                    new Rectangle(0, 0, Math.Max(1, fillW), Height),
                    Theme.AccentGradTop, Theme.AccentGradBot,
                    LinearGradientMode.Horizontal))
                    g.FillRectangle(brush, 0, 0, fillW, Height);
            }
        }
    }
